<Subheading level={2}>1. GitHub アカウントを用意する</Subheading>
<Text className="mt-3">
  初めての場合は <TextLink href="https://github.com/">GitHub</TextLink> にアクセスし、無料アカウントを作成します。メールアドレスの確認が完了したら、プロフィール設定と<Term definition="パスワードに加えてワンタイムコードを求める保護手段。万が一の漏洩時も乗っ取られにくくなります。">2 段階認証</Term>の有効化を済ませると安全です。
</Text>

<Text className="mt-4">
  GUI ツールを使う場合は <TextLink href="https://desktop.github.com/">GitHub Desktop</TextLink> をインストールし、同じアカウントでサインインしておくと、リモート操作をボタン一つで確認できます。
</Text>

<Divider className="my-12" />

<Subheading level={2}>2. リモートリポジトリを作成する</Subheading>
<Text className="mt-3">
  GitHub にログインした状態で右上の <Strong>＋</Strong> → <Strong>New repository</Strong> を選び、次の項目を設定します。
</Text>

<ol className="mt-4 list-decimal space-y-3 pl-6 text-base/7 text-gray-700 dark:text-gray-300">
  <li><Strong>Repository name</Strong>：任意（例：<Code>ai-notes</Code>）</li>
  <li><Strong>Public / Private</Strong>：公開範囲を選択（学習用は Private 推奨）</li>
  <li><Strong>Add a README</Strong> はチェックしなくても OK（後で作成可能）</li>
</ol>

<Text className="mt-4">
  作成すると、リポジトリのトップページに「Quick setup」ガイドが表示されます。ここにローカルと接続するためのコマンドが載っているので、次の章で使用します。
</Text>
<div className="mt-3 rounded-md border border-emerald-200/80 bg-emerald-50 px-3 py-2 text-sm text-emerald-900 dark:border-emerald-300/50 dark:bg-emerald-400/10 dark:text-emerald-100" role="note">
  <p className="leading-6">補足: <Strong>Quick setup</Strong> のコードブロックは HTTPS と SSH の 2 種類が表示されます。最初は HTTPS の URL をコピーすれば OK です。SSH は後で鍵を登録してから使用します。</p>
</div>

<Divider className="my-12" />

<Subheading level={2}>3. 既存のローカルリポジトリを接続する</Subheading>
<Text className="mt-3">
  すでにローカルで Git 管理しているフォルダがある場合は、クリーン（未コミットの変更がない）状態を確認してからリモートを登録します。<Term definition="Git が覚えている接続先のニックネーム。初期設定では origin をメインの同期先として使います。">リモート</Term> は URL に別名を付けたものと考えるとわかりやすいです。
</Text>

```bash
# 現在の状態を確認
git status

# リモートを登録（URL は GitHub の "Quick setup" からコピー）
git remote add origin https://github.com/your-account/ai-notes.git

# 初回の push（main ブランチを公開）
git push -u origin main
```

<Text className="mt-4">
  <Code>-u</Code> オプションを付けると、次回以降は <Code>git push</Code> だけで OK になります。ブランチ名が <Code>master</Code> の場合はそのまま読み替えてください。
</Text>
<Text className="mt-2 text-sm text-gray-600 dark:text-gray-400">
  迷ったら <Code>git remote -v</Code> で登録済みのリモート一覧と URL を確認し、意図した接続先が設定されているかチェックしましょう。
</Text>

<Divider className="my-12" />

<Subheading level={2}>4. リモートとの基本的なやり取り</Subheading>
<Text className="mt-3">
  接続ができたら、以下の3つの操作を覚えれば共同作業の第一歩は完了です。
</Text>

<ul className="mt-4 space-y-4 text-base/7 text-gray-700 dark:text-gray-300">
  <li>
    <Strong>最新の変更を取り込む：</Strong> <Code>git pull</Code>
    <Text className="mt-1">
      他のメンバーが push した変更を自分のローカルに取り込みます。競合が起きた場合は Git ブランチ編で学んだ手順で解決します。
    </Text>
  </li>
  <li>
    <Strong>自分の変更を共有する：</Strong> <Code>git push</Code>
    <Text className="mt-1">
      コミットをリモートへ送ります。初回だけ <Code>git push -u origin main</Code>、2 回目以降は <Code>git push</Code> で大丈夫です。
    </Text>
  </li>
  <li>
    <Strong>新しくリポジトリを取得する：</Strong> <Code>git clone &lt;URL&gt;</Code>
    <Text className="mt-1">
      別の PC で作業したいときや、新しいメンバーが参加するときは GitHub 上の URL を指定してクローンします。
    </Text>
  </li>
</ul>
<div className="mt-3 analogy-card" role="note">
  <p className="leading-6">たとえ: <Code>git pull</Code> はロッカーから最新の資料を持ち帰る動き、<Code>git push</Code> は自分の更新版をロッカーに戻す動き、<Code>git clone</Code> はロッカー全体を丸ごと複製して別の机に置く動きです。</p>
</div>

<Divider className="my-12" />

<Subheading level={2}>5. 誤って公開しないためのチェック</Subheading>
<ul className="mt-4 space-y-3 text-base/7 text-gray-700 dark:text-gray-300">
  <li>
    <Strong>.gitignore</Strong> を確認：API キーや巨大ファイル（モデルやデータセット）が commit されないようにする
  </li>
  <li>
    push 前に <Code>git status</Code> と <Code>git diff</Code> を確認：意図しない差分が含まれていないかを毎回チェック
  </li>
  <li>
    機密情報を誤って push したら <TextLink href="https://docs.github.com/ja/authentication/keeping-your-account-and-data-secure/removing-sensitive-data-from-a-repository">公式手順</TextLink> を参考に早急に履歴を書き換える
  </li>
</ul>
<Text className="mt-3 text-sm text-gray-600 dark:text-gray-400">
  <Term definition="Git で追跡から外したいファイルを列挙する設定ファイル。ここに書くとコミット対象から外せます。">.gitignore</Term> を整えておくと、鍵情報や巨大ファイルがコミットへ紛れ込みにくくなります。
</Text>

<Divider className="my-12" />

<Subheading level={2}>.gitignore で守るもの</Subheading>
<Text className="mt-3">
  <Term definition="Git で追跡から外したいファイルを列挙する設定ファイル。ここに書くとコミット対象から外せます。">.gitignore</Term> に記載したファイルは、ローカルで編集してもリモートには送られません。パスを記載し忘れると、API キーや環境設定、巨大な学習データがそのまま GitHub に載ってしまう可能性があります。
</Text>
<Text className="mt-3 text-sm text-gray-600 dark:text-gray-400">
  うっかり秘密情報を push すると、記録をさかのぼって完全に消すのが大変です。公開リポジトリなら不特定多数に漏洩し、Private でもメンバー全員に共有されてしまいます。初期段階で <Code>.env</Code> や <Code>*.log</Code> など「公開したくないもの」を必ず列挙しましょう。
</Text>
<Text className="mt-3 text-sm text-gray-600 dark:text-gray-400">
  迷ったら <Code>git status</Code> で差分を確認し、コミットに含めたくないファイルが赤字で表示されていないかチェックしてください。気付いた時点で <Code>.gitignore</Code> に追記すれば、以降のコミットには含まれません。
</Text>
